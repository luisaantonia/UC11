# SOFTWARE DELIVERY 

----------

**Requesitos funcionais**

1.  Cadastro de usuários<br> 
2.  Login com e-mail e senha<br>
3.  Validação de dados obrigatórios<br>  
4.  Atualização de dados do usuário<br>
5.  Exclusão de conta<br>  
6.  Cadastro de endereços<br>
7.  Listagem de endereços<br>  
8.  Criação de pedidos<br>  
9.  Visualização de pedidos<br>  
10. Cancelamento de pedidos<br> 
11. Controle de status do pedido<br>  
12. Rastreamento de entrega<br>  
13. Login de entregadores<br>  
14. Aceite de entregas<br>  
15. Início da entrega<br>  
16. Finalização da entrega<br>  
17. Avaliação de entrega<br>  
18. Histórico de entregas<br>  
19. Recuperação de senha<br>  

---------------

**Não funcionais**

 1.  Disponibilidade de 99%  
 2.  Tempo de resposta < 2s  
 3.  Responsividade (mobile)  
 4.  Segurança com criptografia  
 5.  Suporte a múltiplos usuários  
 6.  Logs de auditoria  
 7.  Backup automático  
 8.  Usabilidade adequada  
 9.  Escalabilidade  
 10. Integridade dos dados  
 11. Autenticação segura  
 12. Conformidade com LGPD  
 13. Interface amigável  
 14. Facilidade de manutenção  
 15. Documentação técnica

# TESTES 

**Teste do cadastro de usuário**

 1. A pessoa vai fazer um `login`<br>
 2. Vai digitar um nome de `usuário`<br>
 3. Ela vai digitar uma `senha`<br>
 4. E logo após fazer tudo isso vai clicar em `entrar`<br>

-----------

**Teste do movimento de entrega**

1. O pedido é confirmado 
2. Logo após isso o pediso é preparado 
3. Assim que pronto o pedido vai para a entrega 
4. O entregador realiza a entrega para o usuário
5. Assim que entregue o pedido é finalizado

------------

**RESULTATO ESPERADO**

O pedido aparece como `Entregue` no sistema
